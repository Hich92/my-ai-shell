# 📜 CHARTE DE DÉVELOPPEMENT : OS MOONDUST (AI-SHELL)

## 1. ARCHITECTURE & ISOLATION SOUVERAINE

- **MODULARITÉ CRITIQUE** : Tout code métier réside exclusivement dans `/src/modules/`.
- **DÉCOUPLAGE "ISLAND"** : Interdiction stricte d'imports inter-modules. La communication passe uniquement par les services ou hooks partagés dans `/src/lib/`.
- **SOUVERAINETÉ SQL** : Interdiction formelle pour l'IA de modifier le schéma DB (Only Human Access). Utilisation exclusive de requêtes SQL natives via `pool.query`.
- **DRY & CLEAN** : Extraction systématique des composants réutilisables dans `/src/components/shared/`.

## 2. STANDARDS UI : "SHADCN-ONLY" ENFORCEMENT

- **COMPOSANTS NATIFS PROSCRITS** : Interdiction totale d'utiliser des balises HTML brutes (`<button>`, `<input>`, `<table>`, `<card>`, etc.).
- **BIBLIOTHÈQUE @/COMPONENTS/UI** : Utilisation exclusive et systématique des composants Shadcn.
- **PROTOCOLE D'INSTALLATION** : Si un composant Shadcn manque (ex: `table`), l'IA doit stopper la génération, signaler le manque au "Souverain" et attendre l'installation via `npx shadcn-ui@latest add`.
- **DESIGN "B2B PREMIUM"** : Esthétique minimaliste, Tailwind CSS pur, espacements généreux et tons neutres.

## 3. RIGUEUR TECHNIQUE & TYPAGE

- **ZÉRO COMPROMIS TS** : Le type `any` est banni. Utilisation impérative d'interfaces ou de types explicites. Validation des entrées/sorties via **Zod**.
- **CONVENTIONS AGENTS** : Alignement absolu sur les standards définis dans `AGENTS.md`.

## 4. PARADIGME "FUTURE-PROOF" & UX

- **PLACEHOLDERS ACTIFS** : Toute fonctionnalité prévue mais non codée DOIT être présente dans le DOM.
- **STYLE "GHOST"** : Appliquer `opacity-50 grayscale pointer-events-none` sur ces éléments.
- **SIGNALÉTIQUE** : Ajouter un `Badge` (variant: outline) "Bientôt disponible" et un `Tooltip` décrivant l'utilité future.

## 5. PROTOCOLE D'AUDIT & TRAÇABILITÉ

- **TRACE SYSTÉMATIQUE** : Chaque mutation (INSERT, UPDATE, DELETE) doit générer un `AuditLog` dans la table `core.t_audit`.
- **STRUCTURE DES LOGS** : Spécifier le `module_name`, l'`action_type`, la `severity` et inclure les mutations en `JSONB`.
- **CONFORMITÉ** : Avant validation, vérifier la cohérence stricte avec ce fichier.
